# repo

Lightweight workspace for daily ServiceNow investigations and development PoCs.

## What This Repository Is For

- Keep investigation logs reusable and searchable.
- Run small PoCs quickly before full implementation.
- Capture AI-assisted coding prompts and outcomes in a repeatable format.

## Folder Structure

- `docs/servicenow-investigations/`: ServiceNow investigation notes.
- `poc/`: Proof of concept plans, code snippets, and results.
- `test.txt`: Simple test file created during repository bootstrap.

## How To Use

1. Copy a template before starting work.
2. Fill in scope, hypothesis, steps, and result.
3. Commit in small chunks with clear messages.

## Suggested Workflow

1. Create a short-lived branch per topic.
2. Time-box investigation or PoC sessions (30 to 90 minutes).
3. Record outcomes as `Adopt`, `Hold`, or `Reject`.
4. Merge only when notes are understandable by another team member.

## Safety

- Do not store credentials, tokens, or customer-sensitive data.
- Redact ticket IDs or environment details if needed.

## Rollback

If this initialization is not useful, revert it with Git using one commit rollback.
